"서버에 미리 렌더링된 정적인 HTML에 클라이언트의 JavaScript를 연결하여 동적인 기능을 더하는 과정"

## 하이드레이션 과정

1. **서버에서 HTML 렌더링**: 서버에서 웹 페이지의 초기 HTML을 생성하여 클라이언트로 보낸다.
2. **HTML을 클라이언트로 전송**: 이 HTML은 아직 이벤트 리스너나 상태 관리 같은 동적 기능이 없는 상태이다.
3. **클라이언트에서 JavaScript 로딩**: 클라이언트의 브라우저가 HTML과 함께 JavaScript 번들(예: react 코드)을 다운로드한다.
4. **JavaScript를 HTML에 연결**: 클라이언트 측의 JavaScript가 서버에서 온 HTML의 DOM을 제어할 수 있도록 이벤트를 연결하고 상태를 초기화하는 작업이 이루어진다.

## 왜 하이드레이션이 중요한가?

- **빠른 초기 로딩:** 사용자는 서버에서 먼저 렌더링된 콘텐츠를 보게 되어, JavaScript가 완전히 로드되기 전에도 화면의 일부를 볼 수 있다.
- **사용자 경험 개선:** 페이지의 초기 렌더링 속도와 이후의 동적인 상호작용을 모두 잡을 수 있다.
- **검색 엔진 최적화(SEO):** 서버에서 완성된 HTML을 전달하므로 검색 엔진이 페이지 내용을 더 잘 이해할 수 있다.
